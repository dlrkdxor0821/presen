# LibraryPing — UI 기능 정리

## 도서관 회원 UI (`mobile_web`)
- 로그인
- 요청 인식 (음성 / 텍스트 / 터치) — "○○ 책 갖다줘", "화장실 어디야"
- 도서 검색 (제목·저자·키워드) — 대여 가능 / 대여 중 표시
- 도서 상세 보기 (표지, 소개, 청구기호, 서가 위치, 대여 상태)
- 지도 보기 (도서관 내부 지도 — 서가·시설·내 위치, 검색한 책 위치 표시)
- 도서 미리보기 / 독서 보기 (목차·일부 페이지·소개 읽기)
- 도서 이송 / 픽업 요청 (도서 + 받을 위치 지정)
- 내가 부른 로봇 실시간 추적 (현위치 / 도착 예정)
- 추천 (목적·관심분야 기반, 베스트셀러 / 신간)
- 길잡이 호출 (로봇 불러서 안내받기)
- 내 정보 / 대출 현황 (대여 중인 책, 반납일, 대출 이력)
- 대여 연장 / 도서 예약 (대출 중인 책 예약 대기)
- 반납일 · 연체 알림
- 읽고 싶은 책 (위시리스트 / 즐겨찾기)
- 작업 알림 (내 요청 상태: 수락 → 이동 중 → 도착)
- 접근성 모드 (큰 글씨 · 음성 안내)
- 도서관 안내 (운영시간 / 층별 안내), 언어 선택

## 사서 UI (`central_server`)
- 로그인
- 운영 대시보드 (전체 로봇·작업·알림 한눈에)
- 실시간 모니터링 (로봇별 상태 / 배터리 / 현재 작업 / 에러) + 지도에서 로봇 위치 보기
- 작업 지시 (이송 / 분류 / 정리 / 짐꾼 / 파견) + 작업 큐 · 이력 · 진행률 + 특정 로봇 / 랜덤 로봇
- 작업 알림
- 에러 · 작업 성공/실패 로그 + 재시도
- 도서 관리 (도서 등록·수정·삭제, 재고, 위치 / 청구기호)
- 서가 관리 (섹션 종류 변경, 서가별 배치 지도)
- 정리 · 분류 리포트(잘 수행하였는지, 분류결과, 정리결과 등)
- 회원 관리 (회원 정보 + 대출 이력)
- 대여 / 반납 처리
- 통합 검색 (도서 · 위치 · 회원)
- 야간 보안 (운영모드 전환, 침입 알림, 침입 영상 보기)
- 통계 / 리포트 (대출 통계, 작업 통계)
- 설정 (관리자 계정, 로봇 등록 / IP 관리)

## 로봇 QT 터치패널 = libi_gui (`desktop_gui`)
- 관리자 모드 로그인
- 대기 / 홈 (인사, 표정 표시, 순회 중 표시)
- 친밀감 인터랙션 (손인사 · 표정 등)
- 길잡이 (목적지 선택 → 로봇 앞장, 안내 중 지도·남은 거리·상태 표시)
- 검색 (도서 / 시설) → 위치 지도 표시
- 추천
- 작업 / 안내 상태 표시
- 비상정지 
- 에러 동작 기능 (관리자) — 에러 시 현장 수동 조작 (주행 / 팔 / 주변장치), 상태·로그



이송명령 : 서가 -> 특정장소
현재장소에서 -> 서가에 진열하기가 가능한지
           -> 로봇이 자신의 로봇 바구니에 있는 책을 인식하고 해당 서가에 진열하는 시나리오로 가야할 것 같음


도서 이송명령(서가 -> 특정장소)
분류명령(수거함 -> 분류장소 -> 분류행위)
진열명령(현재 로봇 바구니에 있는 책 인식 -> 해당 책이있는 서가에 진열)

이렇게 3가지를 하면 사실상 완벽 
일단 도서이송명령과 분류명령은 확실하게 진행하는 것으로 알고 있음
