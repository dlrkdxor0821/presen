02 Progress · 로봇팔

로봇팔 Robot Arm

🔬
기술조사
  • Perception
    • YOLO — 채택 X (책 구분에 어려움)
    • OCR — 채택 O (향후 신간도서 대응)
    • QR — 채택 X (인식률 너무 낮음)
    • ArUco — 채택 O
  • Depth CAM 사용 여부
    • 미사용 → 최소거리 15cm
    • 더 좋은 방법 채택
  • 모션 제어
    • move l · move j · jog
  • 제어방식
    • 초기: 화면 중앙 정렬 → 고정 픽셀 오프셋
🦾
현재 채택 기술 · 진행
  • ArUco Marker 방식으로 제어
    • rvec·tvec 기반 위치 좌표 계산
  • 카메라–그리퍼 오프셋
    • Hand-eye Calibration
  • One-shot pose grasp
    • 지속 갱신·보정 대신 초기 1회 좌표를 찍고 이동
  • 도서 grasp 성공
    • 성능 개선 중 → 파라미터 수정
🎯
향후 계획
  • 압전센서 적용
    • 파지 성공 여부(Grip Success) 판단
  • 반복적 실험을 통한 파라미터 수정
    • 카메라 Offset
    • 최종 접근 거리
    • 후퇴 거리
  • Place 수행
    • Pick 과정은 기존 PBVS 방식 유지
    • Place 과정은 IBVS(Image-Based Visual Servoing) 기반 제어 적용 검토
      • 기존 구현은 영상 처리와 로봇 제어를 동기식으로 수행하여, 이미지 갱신 → 제어 → 정지 → 이미지 갱신 과정을 반복
      • 이로 인해 로봇팔의 움직임이 끊기고 자연스럽지 않은 문제가 발생
      • 영상 처리와 제어를 비동기적으로 수행하여,
        • 영상은 지속적으로 갱신하고
        • 제어기는 최신 비전 정보를 이용해 연속적으로 제어하는 구조 검토
    • ArUco Marker 없이 Vision 기반 Place 수행
      • 카메라에서 책꽂이 빈칸을 실시간 검출
      • 검출된 위치(이미지 특징)를 지속적으로 제어기에 전달하여 Place 수행

배차 · DispatchAuction · SSI
제약조건
  1. IDLE 상태인 로봇만 배정 가능
  2. 완주 가능한 로봇만 배정 가능
    완주 관문 — 소비% = 거리×1% + 팔동작×0.5% 일 때, 배터리% ≥ 소비%+15% 이어야 배정
Auction 방식

각 로봇이 비용(Dijkstra 거리)을 중앙에 제출 → 최저 비용 낙찰 · 없으면 거절.

🚦
교통관제 · TrafficDijkstra + 노드예약 + DFS
  1. 경로계획 Dijkstra
  2. 노드예약 가려는 노드 잠금(진입 차단)
  3. 데드락 대기 사이클 DFS 감지 → 최저 우선순위 우회(재계산), 나머지 대기
우선순위 사다리 · 높을수록 안 비킴
  1. 완전 막힘(동적) — 막히면 최상위·풀리면 원복
  2. 충전 복귀(CHARGE) — 방전 임박 보호
  3. 작업 수행중 — 오래된 task·낮은 배터리 순
  4. 순회 중 — 제일 먼저 우회

02GUIDE 기능 · 카메라 추가
현상

대응

전방/사용자 인식용 카메라 추가 장착 · GUIDE 화각·장착 위치 설계 · 추종(re-ID) 모델과 연동.

03바구니 모델 선정

대응
