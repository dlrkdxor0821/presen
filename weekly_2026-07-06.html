01CPU 점유율 · 정밀주행 (Pi4)
현상

순수 bringup만 60~80% → ArUco·UDP 영상 송출 시 97%↑ (load 15·19.5FPS). nav2 주행 토픽까지 받으면 추가 악화 우려.

대응

BT로 프로세스 사용/미사용 구간 구분 · perception AI 서버 오프로딩 · 노드 경량화.

02GUIDE 기능 · 카메라 추가
현상

길잡이(SR-11) 수행 시 현재 센서(RPLIDAR 2D + 파지용 D435)로는 전방 사용자·경로 인식 화각 부족.

대응

전방/사용자 인식용 카메라 추가 장착 · GUIDE 화각·장착 위치 설계 · 추종(re-ID) 모델과 연동.

03바구니 모델 선정
현상

바구니 인식용 딥러닝 모델 방식 미확정 (AR-31 Sprint 2부터 진행 중) · grab 방식(AR-49)도 미착수 → 분류·이송 시나리오 블로킹.

대응

인식 모델 후보 비교·선정 → 학습데이터 보강 → 모델 구축·grab 방식 확정.
